# Testing of AI
Evaluating LLMs, ML models, RAG systems, biases, hallucinations

## Trainee
- Understands the difference between deterministic systems and probabilistic AI systems.
- Knows basic terminology: model, training data, inference, prompt, hallucination.
- Can identify AI components in a system under test and explain why classic test oracles may need adaptation.

## Junior
- Executes functional and exploratory tests against AI features using predefined or simple user scenarios.
- Uses reference examples or labeled data when available.
- Identifies and documents AI-specific defects clearly, including hallucinations, unsafe responses, poor grounding, or unstable formatting/behavior.
- Understands basic evaluation concepts and common risks such as inaccurate output, harmful content, or sensitive data exposure.

## Middle
- Designs tests for probabilistic systems using tolerances, rubrics, datasets, or statistical checks where appropriate.
- Tests RAG/LLM features for retrieval quality, relevance, faithfulness, and hallucination behavior.
- Tests data pipelines and regression after model, prompt, retrieval, or configuration changes.
- Covers edge cases, noisy inputs, robustness, fairness concerns, and integration points with the rest of the system.
- Participates in experiments such as A/B comparisons and can interpret task-appropriate metrics.

## Senior
- Designs an end-to-end AI test strategy covering model, data, prompts, retrieval, tools/agents, integrations, and production monitoring.
- Builds or operates automated evaluation pipelines using human rubrics and, where appropriate, LLM-as-a-judge.
- Defines quality gates for release or promotion using task-appropriate metrics and risk-based thresholds.
- Performs adversarial testing and red teaming for prompt injection, jailbreaking, insecure tool use, PII leakage, data poisoning, abuse, and misuse scenarios.
- Includes non-functional concerns such as latency, throughput, token usage, cost, reliability, and drift.

## Expert
- Establishes organization-wide AI assurance practices covering governance, continuous evaluation, auditability, incident learning, and human oversight.
- Aligns AI quality practices with enterprise risk frameworks, applicable standards, and regulation.
- Leads quality strategy for advanced AI systems including agentic and multimodal systems.
- Coaches senior engineers and helps shape cross-functional practices with product, platform, security, data, and legal teams.
